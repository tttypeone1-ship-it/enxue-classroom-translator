# Enxue 课堂双向翻译器 Beta

手机端优先的课堂中英双向句子级翻译网页原型。

第一版用于跑通交互流程：进入前展示 Beta 与隐私说明，选择语言方向后，点击圆形主按钮请求麦克风权限并显示真实音量反馈，再次点击结束，页面继续使用模拟识别、模拟翻译和模拟播放。

## 当前定位

这是一个 Beta 原型，用于验证课堂现场中英双向句子级翻译的手机端交互。当前麦克风只用于权限测试和音量波纹反馈，不会识别真实语音，不会翻译真实语音，不会生成或保存音频文件。

## 文件说明

- `index.html`：页面结构、简化首页、说明弹窗、主翻译界面。
- `style.css`：手机优先的视觉样式，圆形主操作按钮、紧凑课堂工具风格。
- `script.js`：点击开始/结束、麦克风权限测试、真实音量反馈、模拟翻译数据、模拟播放状态。

## 本地预览

这是纯前端静态原型，不需要后端、不需要数据库、不需要 API Key。

可以使用 Codex 内置 `@Browser` 打开项目中的 `index.html` 进行预览。

也可以在项目目录启动本地静态服务后访问页面。

## 第一版已实现

- 简化首页和开始使用按钮。
- 开始使用前的中英文 Beta 与隐私说明弹窗。
- 中文到英文、英文到中文语言方向切换。
- 根据当前语言方向动态切换原文/译文标签、播放按钮、录制按钮和麦克风提示语言。
- 点击开始、再次点击结束的固定圆形主操作按钮。
- 麦克风权限测试和真实音量驱动的轻量波纹反馈。
- 三段式手机控制台布局：顶部标题和方向切换、中间结果区、底部 recorder dock。
- 原文显示区、译文显示区，长文本在卡片内部滚动。
- 播放译文按钮放在译文卡片标题栏中。

## 麦克风测试说明

- 当前麦克风能力只用于权限测试和音量波纹反馈。
- 当前不会识别文字、不会翻译真实语音、不会保存音频。
- 页面不生成录音文件，不上传音频，不保存历史记录。
- 部分手机浏览器可能要求 HTTPS 或 localhost 才允许 `getUserMedia`。
- 如果局域网 http 页面无法调起麦克风权限，可能是浏览器安全策略限制，不一定是代码错误。

## Cloudflare Pages 部署建议

这是无框架纯静态站点。使用 Cloudflare Pages 时可以选择：

- Framework preset：None / No framework
- Build command：留空或 `exit 0`
- Build output directory：`.`
- Root directory：仓库根目录

部署到 `pages.dev` 后，使用 HTTPS 地址在手机浏览器中测试麦克风权限。
