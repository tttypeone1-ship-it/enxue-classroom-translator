# Enxue 课堂双向翻译器 Beta

手机端优先的课堂中英双向句子级翻译网页原型。

第一版用于跑通交互流程：进入前展示 Beta 与隐私说明，选择语言方向后，点击圆形主按钮请求麦克风权限并启动浏览器原生语音识别，再次点击结束，页面使用真实语音识别结果作为原文，译文仍为模拟数据。

## 当前定位

这是一个 Beta 原型，用于验证课堂现场中英双向句子级翻译的手机端交互。当前使用浏览器原生 Web Speech API 进行语音识别，译文为模拟数据。不会翻译真实语音，不会生成或保存音频文件。

## 文件说明

- `index.html`：页面结构、简化首页、说明弹窗、主翻译界面。
- `style.css`：手机优先的视觉样式，圆形主操作按钮、紧凑课堂工具风格。
- `script.js`：点击开始/结束、麦克风权限测试、真实音量反馈、浏览器原生语音识别、模拟翻译数据、模拟播放状态。

## 本地预览

这是纯前端静态原型，不需要后端、不需要数据库、不需要 API Key。

可以使用 Codex 内置 `@Browser` 打开项目中的 `index.html` 进行预览。

也可以在项目目录启动本地静态服务后访问页面。

## 麦克风与语音识别说明

- 当前使用浏览器原生 Web Speech API（`SpeechRecognition` / `webkitSpeechRecognition`）进行语音识别。
- 中文 → English 模式：识别语言 `zh-CN`。
- English → 中文 模式：识别语言 `en-US`。
- 译文区域仍为模拟数据，不接真实翻译 API。
- 不会翻译真实语音，不会生成或保存音频文件。
- 页面不生成录音文件，不上传音频，不保存历史记录。
- 部分手机浏览器可能要求 HTTPS 或 localhost 才允许 `getUserMedia` 和 `SpeechRecognition`。
- 如果局域网 http 页面无法调起麦克风权限，可能是浏览器安全策略限制，不一定是代码错误。

## Web Speech API 兼容性

- 当前使用浏览器原生 Web Speech API，兼容性取决于浏览器。
- iOS / Safari 使用 `webkitSpeechRecognition`。
- Chrome / Edge 桌面端和 Android 端支持较好。
- 不保证所有浏览器可用，不支持的浏览器会显示提示信息。
- 微信内置浏览器兼容性有限，建议使用系统浏览器测试。

## Cloudflare Pages 部署建议

这是无框架纯静态站点。使用 Cloudflare Pages 时可以选择：

- Framework preset：None / No framework
- Build command：留空或 `exit 0`
- Build output directory：`.`
- Root directory：仓库根目录

部署到 `pages.dev` 后，使用 HTTPS 地址在手机浏览器中测试麦克风权限。
